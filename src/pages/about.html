Como o Framework Frontend funciona

Um resumo da arquitetura, fluxo de trabalho e boas práticas para construir produtos com HTML + JavaScript puro.

Visão geral

O Framework Frontend é um kit opinado para montar single page applications leves. Ele aposta em arquivos estáticos, consumo direto de APIs e um CMS embutido que escreve no localStorage. O resultado é um projeto que roda desde um simples python -m http.server até hospedagens robustas.

  • Router baseado em history.pushState + HTML parcial.
  • Componentes em HTML (fetch) ou JS (classe Component).
  • Assets vendors versionados na pasta src/assets/vendor.

Estrutura

src/
├── components/      # Cabeçalho, rodapé, seções reutilizáveis
├── core/            # Router, Component, Skeleton, helpers
├── data/            # Conteúdos mock (blog, cms)
├── pages/           # HTML carregado pelo SPA
├── services/        # ContentService e integrações futuras
└── assets/
    ├── css/         # Temas e overrides
    └── vendor/      # Bootstrap, Swiper, Icons, etc

Ao navegar para /contact, o Core busca src/pages/contact.html, injeta no #root e reexecuta os componentes daquele fragmento. Tudo funciona em conjunto com o CMS, que mantém o conteúdo sincronizado.

Fluxo de desenvolvimento

  1. Clone o projeto e rode um servidor estático local.
  2. Edite os textos via /cms ou diretamente nos arquivos em src/pages.
  3. Para novos componentes, use Component.register('nome') e coloque <div data-component="nome"> no HTML.
  4. Consuma APIs com core.fetchAPI para herdar cabeçalhos e tratamento de erro.

Boas práticas

  • Centralize todas as URLs em src/config/config.js.
  • Use core.showSkeleton enquanto carrega dados.
  • Evite duplicar vendors: use os arquivos em src/assets/vendor.
  • Versione os JSON exportados do CMS para rastrear alterações de conteúdo.
  • Se precisar de múltiplos idiomas, salve diferentes arquivos no CMS e carregue conforme o contexto.

Próximos passos
